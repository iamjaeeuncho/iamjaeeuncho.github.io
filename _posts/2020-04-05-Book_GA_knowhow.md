### 데이터 분석, 어떻게 시작해야 할까요?
1. 처음부터 거창한 인사이트를 얻으려 하지 않고, 분석은 무조건 쉽게 접근하기
2. 분석의 시작은 정확한 데이터 수집. 정확한 데이터가 수집되지 않으면 분석 결과는 신뢰할 수 없음
3. 서비스의 기본적인 UX를 파악하기
4. 퍼널을 설계하고, 가장 중요한 목표 설정(KPI)
5. 분석을 위한 가설(아이디어)를 모으기: 퍼널을 통해 각 단계별 전환율을 확인했다면, 이탈이 많이 발생하는 단계 전환 개선하기위한 아이디어 도출
6. 수집한 가설의 우선순위 정하기
cf. Impact(KPI 지표의 성장에 도움을 주는가), Chance of Success(변화를 통한 개선 가능성), Effort(투입 인력 및 시간 자원)
7. 얻을 수 있는 결과를 금액으로 산출하자: 예를 들면, 전환율이 0.1% 상승할 경우 매출은 1억이 상승될 것으로 예상된다.
> * 전환율 개선을 위한 그로스 해킹 프로세스
> 1. 수집하는 데이터의 정합성을 체크
> 2. 서비스 KPI 선정 및 퍼널을 설계
> 3. 취약 단계를 확인하고, 개선을 위해 AB테스트 진행
> 4. 테스트 결과를 공유하고, 서비스 반영을 결정

### 데이터 분석의 핵심은 전환율 개선이다
- 데이터 분석 방법에 정답은 없습니다. 명확한 근거와 결과를 논리적인 과정으로 도출하면 됩니다. 그래서 데이터 분석에는 스토리텔링 능력이 중요시됩니다.
* AARRR 분석 프레임워크
- Acquisition(획득): 고객이 서비스로 유입되는 단계. 핵심 지표는 앱 다운로드 수, 신규방문자 수
- Activation(활동): 핵심 지표인 전환이 되려면 거쳐야 하는 보조 지표는 회원가입/장바구니 담기 등. 유입 후 이탈 고객 줄이려면 회원가입시 적립금을 추가로 주는 방법등의 활동 단계를 달성할 수 있도록 유도.
- Retention(재방문): 보통 신규 고객 획득보다 재방문 고객의 구매 전환율이나 평균 구매액은 높게 나오고, 획득 비용이 훨씬 적게 듬. '유지가 획득을 이긴다' 신규 고객 획득보다 기존 고객을 놓치지 않는게 중요. 
- Referral(공유): 예를 들어 초대코드와 같이 기존 고객들의 자발적 홍보.
- Revenue(수익): 서비스의 최종 목표 지표로 서비스의 성공과 실패를 보여줌. 쇼핑몰이라면 매출과 인당 평균 결제 금액, 병원은 고객 상담 등. 최초 유입부터 전환까지 전환율을 평균 2% 이상으로 관리해주는게 필요. 중간 이탈하는 고객이 많으면 해당 지점 집중 개선 필요.

### 산업군별 핵심 성과 지표 설계하기
- 쇼핑몰: 재구매율, 구매당 평균 금액(객단가), 앱 다운로드 수, 회원가입 수
- 보험사: 리드 생성 건수, 리드 생성 단가, 보험료 계싼 및 청약 전환율
- 콘텐츠: 체류 시간, 재방문율, 앱 다운로드 수
- 브랜드: 방문자수, 이탈률, 이메일 등록
  * cf. 주의할 점은 핵심 지표가 너무 많이 선정하면 안됨. 지표가 많다는 것은 관리할 수 없다는 의미와 같음.

### 사용자 지표는 고유한 브라우저 쿠키다
- GA가 사용자 인지하는 기준은 쿠키 정보입니다. 쿠키는 기기의 브라우저 단위로 발행됩니다. 동일 기기라도 인터넷 익스플로러와 크롬 브라우저의 쿠키 정보는 다릅니다. 브라우저 쿠키를 삭제하거나 크롬 시크릿 모드로 접속시, 특정 사용자의 방문 이력도 삭제됩니다. GA는 순쿠키 횟수를 카운팅하므로 동일한 사람이 7번 방문해도 1로 카운팅 됩니다. 그래서 다른 분석 도구와 사용자 지표를 비교했을 때, GA가 낮게 나옵니다.  
- 일반적으로 방문자 중 30%는 한 달에 한 번은 쿠키를 삭제하며, GA쿠키는 방문자가 삭제하지 않는 이상 2년 동안 지속됩니다.  
- 만약 브라우저와 관계없이 로그인 사용자를 분석하고 싶다면 GA의 user-ID 기능을 활용. 사용자가 로그인시 고유한 Key값을 받아서 교차 기기 행동 데이터를 GA에서 확인할 수 있습니다.

### 세션 지표, 제대로 알고 쓰자
- 세션은 웹 또는 앱 접속 후 처음으로 히트가 발생하는 순간 카운팅되며, 최초 카운팅부터 30분간 아무런 액션이 없을 경우 종료됩니다. 세션은 랜딩 페이지에서만 카운팅되기 때문에, 만약 A에 랜딩하고 B로 이동 후 종료하면 A 페이지에서만 세션이 카운팅 됩니다.  
  * GA 세션 종료 케이스
  1. 30분 동안 아무런 액션이 없을 때
  2. 자정 밤 12시가 지날 때
  3. 다른 캠페인으로 웹사이트에 랜딩 될 때
<br>
<br>

### 세션 시간이 0으로 나오는 이유
- GA는 마지막 페이지에 머문 시간을 무시합니다. 그래서 대부분 세션 시간이 0으로 확인된 경우는 세션이 1일 때가 많습니다. 마지막 페이지에서 아무리 많은 시간을 머물러도 GA는 마지막 히트 전 시간만 합하여 세션 시간으로 계산합니다.   
- 만약 마지막 페이지 머문 시간을 세션 시간에 포함하려면 GTM을 통해, 일정 시간 체류시 이벤트를 강제로 발생시키는 방법이 있습니다. 그러면 이전 대비 전체 이탈률이 다소 하락합니다.  
  * cf. 특정 페이지 세션이 0, 페이지뷰가 3인 경우
    - 특정 페이지를 조회했는데, 해당 페이지로 랜딩 된 경우가 없을 때. 세션은 웹/앱 접속 후 처음으로 히트가 발생할 때 측정됨.
<br>
<br>

### 이탈률을 낮춰야 전환율이 개선됩니다
- GA에서 말하는 이탈률의 정확한 의미는 '단일 히트 세션'으로, 세션(방문)에서 히트가 1회만 발생했다는 의미입니다.  
- 하지만 프로모션 페이지 내 CTA버튼 클릭, 외부 도메인 연결, 콘텐츠 5분간 읽고 웹사이트 종료 등의 경우도 GA는 이탈한 사용자라고 간주합니다. GTM으로 이탈로 집계되지 않도록 설정이 필요합니다.  
  * cf. 전체 이탈률이 5%로 너무 낮을 때
    - 보통 웹사이트의 이탈률은 20~30%로, 이탈률이 5%라면 정상이 아닐 가능성이 높습니다. 이런 경우, GA 추적 코드가 중복 삽입되어 히트가 2번 발생하는 경우가 대부분.
<br>
<br>

### 전환율은 랜딩 페이지에 달려 있다
- 랜딩 페이지 이탈을 막아야 전환을 높일 수 있습니다. 특정 랜딩 페이지 이탈률이 높다면 해당 페이지 광고 소재를 개선해야 합니다. 유입은 누구나 할 수 있지만 이탈을 최소화해야 합니다.  
- 서비스의 이탈률은 전체 세션을 기반으로 계산됩니다. 이는 각 랜딩 페이지의 이탈률이 전체 이탈률에 영향을 준다는 의미입니다.  
  * cf. 일반적으로 배너가 키워드 광고보다 이탈률이 높아 광고 단가도 키워드보다 낮습니다. 배너 광고만 보고 바로 구매하는 경우는 흔치 않기 때문에, 배너 광고의 성과는 브랜드 키워드 쿼리가 얼마나 증가했는지로 판단해야 합니다.

### 데이터는 쪼개야 인사이트가 나온다
- 세그먼트를 만들 때는 너무 소수를 대표하는 세그먼트를 만들면 안 됩니다. 임의로 '일반/관심/충성/열성'으로 나눠, 낮은 등급 고객을 상위 등급으로 올리기 위해 문제가 무엇인지 파악하는 것만 알면 분석의 절반은 끝난 것이나 다름없습니다.  
- 쇼핑몰에서 가장 중요한 고객은 바로 충성 고객이며 이들의 평균 구매액을 늘리는 것이 마케팅 활동의 핵심이 되어야 합니다. 이들과 유사 행동을 보이는 고객을 구글 애드워즈 광고에 모수로 활용하면 당연히 일반 대비 광고 반응이 높게 나올 것입니다. 예를 들어, 프로모션 페이지 방문자 중 스크롤 50%와 80% 내린 고객에게 다른 배너 광고를 보여주는 광고를 할 수 있습니다.

### 캠페인 링크 생성시 주의점
- 캠페인이란 특정 목적을 가지고 운영되는 마케팅 활동으로, GA에서 수집하려면 링크에 별도 UTM 변수를 붙여줘야 합니다. 링크 뒤에 붙이는 필수 캠페인 변수는 3가지로, 이 중 하나라도 없으면 캠페인 보고서에 데이터가 수집되지 않습니다.
> * (필수) utm_source(소스명) = 출처(Where)
> * (필수) utm_medium(매체명) = 매체 및 광고 형태(How)
> * (필수) utm_campaign(이름) = 목적(What)
> + (추가) utm_term(키워드명) = 유입된 키워드
> + (추가) utm_content(광고 콘텐츠) = 콘텐츠 컨셉
<br>
<br>

### 캠페인 유지 기간이 중요한 이유
- 직접 방문 이전에 캠페인을 통한 방문 이력이 있으면 GA는 Direct 방문이 아닌 캠페인 방문으로 성과를 먼저 부여합니다. 
- GA 캠페인 유지 기간은 기본적으로 6개월로, 속성 설정의 세션 메뉴에서 최소 1분에서 24개월까지로 조정할 수 있습니다. 참고로 네이버 광고 성과 유지 기간은 7~20일이므로, 네이버 의존도가 높을 경우 유지 기간을 1개월로 수정하면 네이버 광고 시스템 성과와 GA간 매체 차이를 줄일 수 있습니다.

### 맞춤형 채널 그룹을 설계하는 방법
- '보기 설정 > 채널 설정 > 채널 그룹 메뉴'에서 특정 소스/매체를 채널로 분류 가능합니다. 다만, 채널 그룹이 상단에 위치할수록 트래픽 분류에 우선순위를 가지게 됩니다.
- 채널은 기본 채널과 신규 채널 그룹이 있는데, 기본 채널 그룹은 생성 시점부터 데이터가 분류되고 신규 채널 그룹은 생성 이전 데이터도 소급 적용되는 장점이 있습니다. 또한 기본 채널 그룹은 데이터 샘플링이 걸리지 않지만 신규 채널 그룹은 샘플링에 영향을 받습니다.  
  * cf. 채널별 데이터 분류 기준 (https://support.google.com/analytics/answer/3297892?hl=ko)

### 구글 오가닉 검색어 확인하는 방법
- 구글에서 검색 광고가 아닌 자연 검색으로 유입되는 경우 키워드 값이 (not provided)로 표현되는데, 이를 확인하기 위해서는 구글 서치 콘솔을 등록해야 합니다. 구글 서치 콘솔에 등록하면 '총 클릭수/총 노출수/평균 CTR/평균 게재순위'를 확인할 수 있지만, 각 키워드별 전환율 및 목표 달성 여부까지는 확인되지 않습니다.  
- 브랜드 인지도가 클수록 브랜드 키워드의 양이 많아집니다. 만약 구글 검색 결과에 노출이 많이 되고 게재 순위가 낮은 경우라면 해당 키워드는 CPC 단가를 높여서 클릭 유도하는 전략이 필요합니다.

### 필터 설계에 관한 다양한 팁
- 필터는 정확한 데이터를 수집하기 위해 적용하며, 필터는 '보기 설정' 메뉴에서 조작 가능합니다. 참고로 필터는 적용 이후 누적 데이터에만 영향을 받을 뿐, 이전 데이터에는 적용이 되지 않습니다. 또한 필터가 다수일 경우 필터가 생성되는 시간 순서대로 적용이 됩니다. 
  * 필터가 적용되는 경우
    - 업무 유관자(내부 직원 및 대행사) 및 봇(Bot) 트래픽 제외
    - 특정 도메인 혹은 기기별 트래픽 보기를 만들 때
    - 수집되는 데이터 패턴을 조작할 때

### 웹사이트 내부 검색어 확인 방법
- 고객의 관심사를 즉각적으로 확인할 수 있는 0순위 데이터는 내부 검색어입니다. 계절 및 이벤트의 영향을 많이 받는데, 내부 검색어 트렌드 추이와 검색엔진에서 유관 키워드 검색 추이를 살피며 고객 니즈에 빠르게 대응해야 합니다.  
- 보통 링크 주소에 SearchQuery 항목으로 보여지는데, 검색해도 변수가 확인되지 않는 경우는 개발자가 웹사이트 개발시 검색어 변수를 숨김 처리했기 때문입니다.(Post 방식) 이 경우에는 GTM으로 가상 페이지뷰를 만들어, 해당 페이지에서 검색어 변수 입력시 검색어 보고서에서 해당 데이터를 보여주도록 설정해야합니다.  
- 검색 기능 경험자 세그먼트로 비교해보면 일반적으로 내부 검색 기능을 이용한 사용자는 그렇지 않은 사용자 대비 평균 전환율과 구매액이 높습니다. 그러므로 '추천 검색어'와 '연관 검색어'로 방문 고객들이 검색을 하도록 유도하는 전략이 필요합니다.

### 기획에서부터 분석을 고려해야 한다
1. 페이지 제목과 경로(url)는 분석의 중요한 재료다: 장바구니 페이지 제목 '장바구니', 경로 URL에 'cart'라는 단어를 붙이는 경우가 많은데, 이는 SEO 측면에서 검색 순위가 상위에 위치하도록 도움을 줍니다.
2. 배너 이미지 태그에 Alt(대체 텍스트) 속성 값을 붙여주자: 랜더링 실패시 이미지 대신 문자열이 나오도록 설정된 것이므로, 웹 접근성 준수를 위해서도 반드시 필요하며, 또한 alt 값은 방문자가 이미지 클릭시 어떤 그림을 클릭했는지 데이터를 수집할 수 있게 합니다.
3. 동일한 서비스 확장이라면 서브 도메인을 생성하자: 만약 도메인이 다르면 GA상에서는 다른 방문자로 인식하기때문에 분석에 많은 작업이 소요.

### 코호트 분석을 제대로 활용하는 방법
- 동질 집단의 개념은 특정 기간 내에 공통된 특징을 가진 사용자 그룹으로, 예를 들어 장바구니에 상품을 추가한 여자 또는 방문 후 이탈하지 않은 남자 등이 있습니다. 코호트 분석 포인트는 단순 재방문이 아닌 특정 액션을 한 고객의 재방문 데이터를 체크해야 한다는 것 입니다.  
- 주의할 점은 데이터 조회할 때 서비스 도메인에 맞는 동질 집단 크기를 선택해야 합니다. 예를 들어, 여행 서비스면 주간/월간 단위, 웹툰 서비스의 경우 일별/주간 단위로 데이터를 조회해야 합니다.  
- 코호트 분석 보고서의 데이터는 데이터 양이 많을 경우 샘플링 때문에 결과 신뢰가 힘들고 제대로 해석하지 않으면 잘못된 액션을 도출할 가능성이 높아 자주 활용하는 보고서가 아닙니다. 때문에 여러 세그먼트를 적용해 다양한 관점에서 데이터 조회 작업이 필요합니다.

### 직접 유입 트래픽이 높은 이유는 무엇일까
- GA 직접 유입 트래픽은 이전 페이지 경로가 없다는 것을 의미합니다. 하지만 유입시 분류되지 않는 트래픽은 모두 직접 유입으로도 잡힙니다. 직접 유입 트래픽이 모든 트래픽 중 80% 이상을 차지한다면, 랜딩시 리다이렉션 과정이 있는지 확인해야 합니다.  
  * 직접 유입 트래픽으로 분류되는 경우
    - 웹문서 혹은 PDF 다운로드 링크 클릭시
    - 랜딩 링크에 캠페인 매개변수를 붙이지 않을 때
    - 리퍼러 정보가 유실되는 경우
    - 서버 사이드가 아닌 클라이언트 사이드 리다이렉션 발생시
    - HTTPS 프로토콜에서 HTTP 프로토콜로 이동시
    - GA 코드 미설치 페이지에서 설치 페이지 이동시
    - 메시징 서비스를 통한 링크 클릭

### 카테고리 행동 패턴 분석
콘텐츠 그룹은 페이지를 그룹 단위로 묶어서 고객의 행동 패턴을 분석하는 방법입니다. 콘텐츠 그룹을 통해 데이터를 조회하면 페이지가 아니라 특정 그룹 단위로 행동 패턴을 조회할 수 있습니다. '보기> 콘텐츠 그룹 메뉴'에서 설정하고, '행동 > 모든 트래픽'에서 확인합니다.
그룹에 어떤 항목을 넣어야 할지 감이 잡히지 않으면, GNB 메뉴를 그대로 넣습니다. 주의할 점은 콘텐츠 그룹에서 조회되는 'not set' 항목의 트래픽 비중을 3% 내외로 낮게 유지해야 합니다.

### 샘플링 해결책
샘플링은 원본 데이터의 일부를 가지고 전체 결과를 추산하는 방법입니다. 샘플링이 적용되면 GA에서는 오른쪽 상단에 샘플링 비율이 표기됩니다.
1. 유료 버전 사용시 행동 흐름 보고서를 제외하고는 샘플링에서 거의 자유롭습니다. 
2. 높은 샘플링 정확도 적용하기: 샘플링되는 데이터 크기가 클수록 결과는 원본에 가깝습니다. (Greater precision 선택)
3. 샘플링 가능성 애초에 차단하기: 세그먼트나 맞춤 보고서로 데이터 조회시 샘플링에 걸리므로, 자주 조회하는 경우에는 해당 데이터 조회만을 위한 보기를 별도로 생성하면 됩니다.
4. 애널리틱스 캔버스 3rd Party 도구 활용: 데이터 분활해서 조회하면 샘플링이 걸리지 않습니다. 이를 도와주는 도구가 'Analytics Canvas'이나 유료이고 어렵고 한글 지원이 되지 않습니다.

### 구글 태그 매니저
- 태그는 What, 트리거는 When: 태그(Tag)는 데이터 혹은 스크립트를 담기 위한 그릇, 트리거(Trigger)는 태그가 언제 실행되는지를 정의합니다. 태그가 실행되려면 트리거가 있어야 합니다.
- 태그와 트리거 설계 완료시, GTM 미리보기 모드와 GA 실시간 보고서에서 수집되는지 체크: 여러 브라우저 및 PC, Mobile 기기에서 확인 후 레이아웃 상 오류 없는지 체크
- 링크 다운로드, 동영상 재생, 마우스 스크롤 등 이벤트 히트 데이터는 별도로 추적해야 합니다.
cf. GA 이벤트 구성요소 : 카테고리(ex. Video), 액션(ex. Play), 라벨, 값(ex. 영상명) 


### 가상 페이지뷰 수집이 필요한 이유
- 가상 페이지뷰란 실제로는 존재하지 않는 가상의 페이지뷰로, 페이지가 바뀌었음에도 불구하고 페이지 URL이 변하지 않아서 데이터 추적이 어려울 때 가상의 페이지뷰를 생성해서 카운팅하는 역할입니다.
- 가상 페이지뷰 생성하기 (P.167)
1. Track type = Page View
2. More Setting에서 page 선택 후, '/virtual'로 시작하는 주소 생성


### GTM으로 페이스북 기본 픽셀 
- 페이스북 픽셀을 웹사이트에 심으면, 페이스북/인스타그램 광고시 웹사이트에 방문한 사람들에게만 광고를 노출하는 리타겟팅 광고 집행이 가능합니다. 구매 가능성이 높은 고객에게 맞춤형 광고를 제공하기 위해 필요합니다. 예를 들어 '검색/조회/장바구니/전환' 등 특정 액션을 했다는 것을 광고 모수로 쌓을 수 있습니다. 다만, 전환 픽셀은 특정 이벤트 시점에 실행되야하고, 항상 모든 페이지에서 실행되는 기본 픽셀보다 뒤에서 실행되어야 합니다.


### Q&A
* 네이버 페이 성과 추적
- 네이버 페이 클릭과 동시에 웹사이트를 떠나기 때문에 파악 어려움. 단, 클릭되는 상품명과 가격은 GTM으로 추적가능하나 구매까지는 추적 불가능.


* 블로그 유입시 전체 URL 보기
- 블로그를 통해 유입된 경우, GA는 해당 블로그 링크를 모두가 아니라 일부만 보여줌. 어떤 블로그에서 온건지 보고 싶을 때는 별도의 필터를 적용해야 함. 
- 'Full Referral'이라는 필터 생성 → 'Custom' → 'Advanced' → 'Field A -> Extract A'에서 'Referral' 선택 및 '(.*)' 입력 → 'Output To -> Constructor'에서 'User Defined' 선택 및 '$A1' 입력.


* 신규와 재방문 개념
신규방문자는 추적 코드가 설치되고, 처음 확인된 쿠키 ID. 재방문자는 이전에 수집된 GA 쿠키 ID와 일치하므로 쿠키를 삭제하게 되면 신규 사용자로 인지됨.


* SEO 개선 방법
- SEO는 검색엔진 최적화로, 검색 엔진이 웹사이트를 쉽게 검색할 수 있도록 작업해야 한다는 의미로 광고보다 더 강력. 가장 중요한 요소는 '페이지에 담긴 내용'과 '제목'.
1. 이미지보다는 텍스트
2. 이미지 넣을 경우에는 Alt 값을 넣어 설명
3. 태그를 넣어서 키워드 검색 시 노출될 수 있도록





* Zum 검색엔진 트래픽 Organic으로 변경
- 속성 설정에서 오가닉 검색 소스에 Zum 검색엔진 추가시, Referral에서 Organic으로 수정됨.
