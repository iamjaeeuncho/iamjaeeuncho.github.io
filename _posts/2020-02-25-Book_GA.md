### 각 역할별 분석 포인트
  - 마케터: 어떤 상품을, 어떤 대상에게, 어디에서 진행할 것인이 확인해 마케팅 효율 극대화
  - 기획자: 사용자가 홈페이지를 어떻게 사용하는지 확인해, 사용자의 요구를 니즈를 홈페이지에 반영하도록 개선해야함 
    - ex. 시작페이지 접속 후 바로 회원가입 페이지, 장바구니, 위시 리스트 버튼을 누르는지. 또는 A페이지 다음에 B페이지를 접속하는지. 접속 후 50%가 아무것도 하지 않고 홈페이지를 떠나는지
  - 디자이너: 사용자가 어떤 버튼을 많이 클릭하는지, 어떤 부분이 사용성이 안 좋은지 확인해 디자인 개선.
    - ex. 무료 가입하기 vs 회원 가입하기 버튼 비교
  - CEO: DAU, MAU
<br>
<br>

### 보고서 비교
  - 일정 비교: 날짜의 비교대상 탭 적용 (ex. 11월 vs 12월)
  - 측정 기준 비교: 측정항목 선택 (ex. 사용자 vs 페이지뷰 수)
<br>
<br>

### 가설과 검증
  - 의문을 제기하는 것을 가설 설정이라 하고 이를 확인하는 것을 가설 검증이라 함.
  - ex. 사용자 수가 늘어나면 페이지뷰 수도 늘어나지 않을까? -> 가설 설정
  - cf. 구매 완료 전환율을 활용해 구매 완료 1000건이라는 목표를 달성하기 위해서는 어느 정도의 세션이 장바구니 페이지에 유입되어야 하는지 추정도 가능.
<br>
<br>

### 잠재고객 보고서 주요 항목
  - 사용자 (New Visitor / Returning Visitor)
  - 세션: 사용자의 홈페이지 사용을 나타내는 단위.
    - GA는 사용자 접속부터 마지막 행동의 30분 뒤를 한 덩어리로 묶어 세션(사용)으로 측정. (ex. 사용자A가 11시에 홈페이지에 접속해 11시 15분에 떠나면, 11:00~11:45까지가 하나의 세션. 만약 11:40에 다시 접속한다면 해당 세션 연장되어 12:10까지가 하나의 세션이 됨
  - 사용자당 세션 수: (ex. 사용자A가 오전, 오후 한번씩 홈페이지 사용하면 사용자 당 세션 수는 2)
  - 페이지수: 전체 사용자의 페이지 조회수
  - 세션당 페이지수: (ex. 사용자A가 1회 사용했을 때 10페이지를 보면, 세션당 페이지수는 10)
  - 평균 세션 시간: (ex. 사용자 2명 중 한 명이 홈페이지를 2분 사용하고 나머지 한 명이 4분 이용하면 평균 세션 시간은 3분)
  - 이탈률: 한 페이지만 보고 나가는 사용자의 비율
<br>
<br>

### 진단법 예시
  - 재방문 사용자가 지나치게 작을 때, 최근 한 달 동안 방문 이력이 없는 가입자에게 재방문 이메일 발송하는 등 사용자 재방문 유도 방안 연구 필요
  - 이탈률이 50%로 높을 때 → 사용자가 원하는 정보를 잘 전달하고 있는지 혹시 우리가 알리고 싶어하는 정보만 제공하고 있는지 확인
  - 사용자 평균 4페이지 사용 → 사용자의 관심을 4페이지 이내에 사로잡아야 한다는 의미.
  - 사용자 한 페이지당 약 36초 사용 → 사용자가 상품을 대략적으로 확인한다는 의미로, 사용자가 큰 관심을 기울이지 않아도 상품의 특장점 등을 쉽게 파악할 수 있도록 효율적 정보 제공 고려 필요.
<br>
<br>

### 사용자수 추정하기
  1. 일정 설정: 예를 들어 기본 일정 8.28-29 vs 비교 일정 8.21-27
  2. 탐색 그래프 데이터 확인: 추이와 일치하는지 여부 확인
  3. 추정 확인하기: 9월 4일에 추정한 사용자 맞는지 확인 (그래프가 일치하는지 여부)
  ∴ 서비스나 홈페이지 변화 없는 이상 사용자 행동이 대부분 비슷하기 때문에 두 개의 비교 그래프 거의 일치.
<br>
<br>

### DAU, MAU 파악하기
  - 사용자 수는 서비스 분석의 가장 기본이 되는 지표. 사용자 수가 늘어야 페이지뷰 수, 가입자 수, 구매자 수가 늘거라 예상하기 때문에.
  - MAU(Daily Average Users): 한달 동안 홈페이지에 접속한 순 사용자 수
  - DAU(Daily Average Users): 하루 접속한 사용자 수. 보고서 내보내기로 확인 가능. 
  - cf. MAU ≠ DAU의 총합 
<br>
<br>

### 사이트 콘텐츠 보고서 항목
  - 페이지뷰 수: 해당 페이지 조회 수
  - 순 페이지뷰 수: 중복 제외한 순 조회 수 (ex. 사용자가 같은 페이지 10번봐도 1번만 순 페이지뷰로 측정)
  - 평균 페이지에 머문 시간
  - 방문 수: 해당 페이지로 홈페이지를 방문(시작)한 수
  - 이탈률: 해당 페이지로 홈페이지를 처음 방문해 해당 페이지만 보고 나가는 비율. (= 다른 페이지 이동하지 않고 해당 페이지만 보고 나가는 경우)
  - 종료율: 해당 페이지에서 홈페이지 종료한 비율
  - 페이지 값: 페이지에 설정한 현금 가치의 합. (ex. 해당 페이지값을 1000원으로 설정한 경우 페이지뷰 수가 2일 때 페이지 값은 2000원)
<br>
<br>

### 사이트 콘텐츠 보고서 읽기
  - 페이지뷰 & 순페이지뷰수: 페이지 뷰는 중복 페이지뷰 수(새로고침, 재방문 등)을 포함. 만일 중복 페이지뷰 수를 제외한 데이터를 원한다면 순 페이지뷰 수 확인
  - 평균 페이지에 머문 시간: 평균 페이지에 머문 시간이 55초라면, 사용자의 다음 행동을 55초 내에 유도해야 함.
  - 방문 수: 전체 사용자의 50%가 /home에서 홈페이지 이용 시작. /home 페이지를 이용하는 사용자가 많으므로 사용자의 요구(Needs)에 부함하는 콘텐츠가 표시되도록 노력. 쇼핑몰 성격이 강하므로 많이 팔리는 상품, 인기 있는 상품이 돋보이도록 홈페이지 구성하는 것 고려 필요.
  - 이탈률: /home 페이지 이탈률 45%. 사용자의 요구에 부합하는 콘텐츠를 표시해 /home 이외의 다른 페이지 사용할 수 있도록 유도 필요
  - 종료율: 이곳저곳 페이지 살피다 /home 페이지를 마지막으로 홈페이지 꺼버리는 비율. 이탈률은 /home에 접속해 /home 페이지만 보고 꺼버리는 비율.
  cf. 이탈률과 종료율로 문제 파악하기
    - 홈페이지에 접속한 사용자가 한 페이지만 보고 떠나는 경우 이탈률 증가. 시작 페이지에서 이탈률이 높게 나타난다는 것은 사용자가 홈페이지를 더 볼 것이 없다고 판단했다는 의미로, 사용자가 원하는 정보가 없는지 점검해야 함.
    - 사용자가 특정 페이지에서 홈페이지를 떠나면 종료율 증가. 회원가입 페이지에서 종료율이 높다는건 사용자가 회원가입을 어렵게 느끼고 있는건 아닌지 점검해야함. 
<br>
<br>

### 종료율로 문제 페이지 발견하기
  종료율이 발생하는 것은 당연하지만 문제는 어느 페이지에서 발생했는가가 중요
  1. 종료율 필터링하기: 모든 페이지 종료율을 살펴볼 수 없으므로 평균 데이터 기준 불필요한 데이터 제외
    - GA상의 평균 종료율을 기준으로 삼고, 종료율 초과 데이터만 남도록 필터링 (고급 검색, 검색조건: 종료율, 초과, 25)
  2. 페이지뷰 수 필터링하기 (고급검색, 검색조건: 페이지뷰 수, 초과 500 (임의) 적용)
  3. 종료율 점검하기: 필터링 이후에는 사용자가 많이 접속함에도 종료율이 높은 페이지들만 남음
  ※ 종료율이 높은 이유: 원하는 상품이 없기 때문에 - 사용자가 다른 상품에도 관심 두도록 유도 필요
  ※ 회원가입, 로그인, 결제 페이지는 이탈률과 종료율을 주기적으로 점검하는 것이 필요
<br>
<br>

### 구매 확률이 높은 사용자 알아보기
  - 방문자가 비구매자에 가까운 사용 패턴을 보이는지 구매자의 사용 패턴을 보이는지 확인하면 구매 확률이 높은 사용자를 구분할 수 있음. 
  1. 구매자와 비구매자 세그먼트 만들기: 주문 완료 페이지에 접속했느냐를 기준으로 세그먼트 만듬
  2. 구매자와 비구매자 세그먼트 비교하기: 비구매자가 구매자보다 많은 것은 당연하나 대신 사용자당 세션 수, 세션당 페이지수, 평균 세션 시간이 중요
  3. 구매자와 비구매자의 사용 패턴 활용하기
  ex. 구매자: 사용자당 세션 수 1.03, 세션당 페이지수 22.09, 평균 세션 시간 14분 59초
  비구매자: 사용자당 세션 수 1.14, 세션당 페이지수 3.49, 평균 세션 시간 2분 5초
  -> 위의 데이터를 기반으로 구매자와 비슷한 방문 패턴을 보이는 방문자에게 쿠폰 증정시 매출 상승 가능성 높음.
<br>
<br>

### 구매자 페이지 이용 순서 확인하기 for 구매 과정 개선
  1. '잠재고객 > 사용자흐름' 메뉴에서 구매자 세그먼트 적용하기
  2. /ordercompleted.html(구매과정 완료) 노드 클릭 - 여기를 통과한 트래픽 조사 클릭
  3. /ordercompleted.html(구매과정 완료) 노드 역추적 - 화면 좌측 + 단계 클릭
  4. 구매흐름확인하기 - 최적은 '장바구니 -> 로그인 -> 구매과정'

### 이벤트 보고서 항목
  - 총 이벤트 수
  - 순 이벤트 수: 중복 제외 이벤트 수. 10번 발생시키면 중복 제외한 1번만 순 이벤트 수로 측정
  - 이벤트값: 이벤트에 설정한 가상의 현금 가치의 합. (ex. 회원가입 이벤트값을 1000원이라고하면 2번하면 2천원)
  - 평균값: 이벤트 값을 총 이벤트 수로 나눈 값
<br>
<br>

### 우수 참여 고객 설정 (Engaged Users)
  1. 관리 > 목표 설정: 세션당 페이지수/조회 화면수 > 10
  2. 우수 참여 고객 세그먼트 만들어서 적용
  3. 비구매자/구매자/우수 참여고객 비교 -> 구매 가능성이 높은 사용자 그룹에게 할인 쿠폰 제공

### 가상 페이지뷰
  - 실제로는 주소가 없는 페이지에 가상으로 페이지 주소를 부여해 실제 페이지처럼 분석할 수 있도록 한 기능
  - 장점: ① 이벤트 데이터보다 다양한 정보 확인 가능, ② 유입경로 시각화 보고서 확인 가능
  - 단점: ① 홈페이지 전체 페이지뷰 수치가 실제보다 높게 표시 ② 뒤로가기, 닫기를 해도 페이지에 머물고 있다고 인식함. 페이지 정보를 부정확하게 만듬.
  - cf. 로그인 버튼 클릭률 계산: 순 페이지뷰 수가 1000, 가상 페이지 나온 횟수가 200 이므로 클릭률은 20%
<br>

### 채널 분석
  - 획득 > 전체 트래픽 > 채널
  (ex. 현재 트래픽의 40% 가량이 다이렉트 채널에서 발생. 이는 지난주보다 2% 높아진 수치)
  - Organic Search: 무료 검색 키워드 트래픽. 구글이나 네이버 검색을 통해 접속할 경우
  - Paid Search: 유료 검색 키워드 트래픽. 구글 검색 광고를 통해 접속할 경우. 국내 검색 광고의 경우 Paid Search가 아니라 Organic Search에 포함.
  - Direct: 홈페이지 직접 접속 트래픽. 직접 주소 입력 또는 즐겨찾기
  - Referral: 다른 곳에서 링크를 클릭해 접속하는 트래픽. 커뮤니티 사이트 등에서 홈페이지 링크 클릭해 접속할 경우
  - Social: 소셜 네트워크에서 접속하는 트래픽. 유튜브, 페이스북 접속시
  - Email: 이메일을 통해 접속할 경우. 사용하는 이메일 프로그램에 따라 안잡힐수도
  - Affiliates: 제휴 광고 트래픽
  - Display: 배너 광고등의 디스플레이 광고 트래픽
  - CPC: 클릭형 광고 트래픽

### 리퍼러 분석하기
  - '획득 > 전체 트래픽 > 추천' 탭
  - 링크를 클릭해 접속 > 마우스 오른쪽 버튼 클릭 후 검사 > Console > 코드 입력란에 document.referrer 입력
<br>
<br>

### 고객ID를 로그인 ID로 표시하기
  - 관리 > 추적정보 > User-Id > 정책 동의를 해제에서 설정으로 변경 > 세션 통합을 설정으로 변경 > 만들기
<br>

### 동질 집단 분석하기 (cf. 동질 집단 분석 = 코호트 분석 Cohort)
  - 잠재고객 > 동질 집단 분석
  - 한 달 동안 사용자 수가 지속적으로 감소하고 있을 때, 일별/주별/월별 사용자가 어떻게 감소하는지 확인하고 감소 예상되는 시점에 미리 대응
  -  ex. 유튜브 접속 세그멘트 적용시, '유튜브를 통해 접속한 사용자의 유지율' 확인 가능: 단순 페이지뷰수가 많은게 중요한것이 아니라 시간이 지남에 따라 어떻게 감소하는가가 더 중요
