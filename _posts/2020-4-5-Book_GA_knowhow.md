### 데이터 분석, 어떻게 시작해야 할까요?
1. 분석은 무조건 쉽게 접근하기: 처음부터 거창한 인사이트를 얻으려 하지 않고, 아이디어를 모으고 퍼널 설계를 통해 데이터 흐름을 파악하다 보면 개선을 위한 인사이트는 자연스럽게 도출
2. 분석의 시작은 정확한 데이터 수집이다: 정확한 데이터가 수집되지 않으면 분석 결과는 신뢰할 수 없음
3. 서비스의 기본적인 UX를 파악하기: 직접 서비스 사용하며 서비스의 기본 기능이 제대로 작동하는지 확인. 기본을 만족시키지 못하는 순간 고객은 실망하고 떠남.
4. 퍼널을 설계하고, 데이터에 애정을 갖기: 분석을 할 때 가장 중요한 것은 목표 설정(KPI)
5. 분석을 위한 가설(아이디어)를 모으기: 퍼널을 통해 각 단계별 전환율을 확인했다면, 이탈이 많이 발생하는 단계 전환 개선하기위한 아이디어 도출
6. 수집한 가설의 우선순위 정하기: 가설을 액션으로 수행하기위해 필요한 인적 자원과 기간이 얼마나 소요되는지, 목표에 영향을 줄 수 있는지 등을 체크. 이를 수치화해서 우선순위가 높은 가설부터 검증. 검증은 데이터를 집요하게 탐색하는 과정. 평균 데이터로는 인사이트를 도출할 수 없으므로, 성별/기기/연령대 등으로 데이터를 나누고 각 집단별 흐름이 어떠한지 확인.
cf. Impact(KPI 지표의 성장에 도움을 주는가), Chance of Success(변화를 통한 개선 가능성), Effort(투입 인력 및 시간 자원)
7. 얻을 수 있는 결과를 금액으로 산출하자: 이해 관계자를 설득하는 작업은 굉장히 힘든 과정입니다. 이럴 때 바로 데이터에 근거한 분석 결과가 필요합니다. 액션을 통해 얻을 수 있는 예상 결과를 금액으로 산출하시기 바랍니다. 예를 들면, '전환율이 0.1% 상승할 경우 매출은 1억이 상승될 것으로 예상된다.'와 같은 식입니다. 분석이 돈이 된다는 사실을 경험한 조직은 데이터의 힘을 믿게 됩니다. 데이터 기반 서비스를 운영하는 조직 문화는 하루아침에 이뤄지지 않습니다.
조화가 잘 되는 팀을 보면 상대방의 주장과 근거가 나보다 나을수 있다는 겸손함을 가진 팀원이 많은 것을 발견합니다
> * 전환율 개선을 위한 그로스 해킹 프로세스
> 1. 수집하는 데이터의 정합성을 체크
> 2. 서비스 KPI 선정 및 퍼널을 설계
> 3. 취약 단계를 확인하고, 개선을 위해 AB테스트 진행
> 4. 테스트 결과를 공유하고, 서비스 반영을 결정

### 데이터 분석의 핵심은 전환율 개선이다
데이터 분석 방법에 정답이란 없습니다. 명확한 근거와 결과를 논리적인 과정으로 도출하면 됩니다. 그래서 데이터 분석에는 스토리텔링 능력이 중요시됩니다. 머릿 속에 생각한 결과를 하나의 스토리로 만들 수 있어야 합니다.  
AARRR 분석 프레임워크: 사용자가 처음 서비스에 유입될 때부터 전환에 이르는 과정을 단계별 핵심 지표로 설계하는 것을 의미.  
- Acquisition(획득): 고객이 서비스로 유입되는 단계. 핵심 지표는 앱 다운로드 수, 신규방문자 수
- Activation(활동): 핵심 지표인 전환이 되려면 거쳐야 하는 보조 지표는 회원가입/장바구니 담기가 있습니다. 유입 후 이탈 고객 줄이려면 회원가입시 적립금을 추가로 주는 방법등의 활동 단계를 달성할 수 있도록 유도.
- Retention(재방문): '유지가 획득을 이긴다' 신규 고객 획득보다 기존 고객을 놓치지 않는게 중요. 보통 재방문자의 구매 전환율이나 평균 구매액이 일반 유저 대비 훨씬 높게 나오며, 신규 고객 획득보다 재방문 고객 획득 비용이 훨씬 적게 듬.
- Referral(공유): 예를 들어 초대코드와 같이 기존 고객들이 자발적 홍보.
- Revenue(수익): 서비스의 최종 목표에 도달했는지 나타내는 지표로 서비스의 성공과 실패를 보여줌. 쇼핑몰이라면 매출과 인당 평균 결제 금액, 병원은 고객 상담 등. 최초 유입부터 전환까지 전환율을 평균 2% 이상으로 관리해주는게 필요. 중간 이탈하는 고객이 많으면 해당 지점 집중 개선 필요.

### 산업군별 핵심 성과 지표 설계하기
- 쇼핑몰: 재구매율, 구매당 평균 금액(객단가), 앱 다운로드 수, 회원가입 수
- 보험사: 리드 생성 건수, 리드 생성 단가, 보험료 계싼 및 청약 전환율
- 콘텐츠: 체류 시간, 재방문율, 앱 다운로드 수
- 브랜드: 방문자수, 이탈률, 이메일 등록
cf. 주의할 점은 핵심 지표가 너무 많이 선정하면 안됨. 지표가 많다는 것은 관리할 수 없다는 의미와 같음.

### 사용자 지표는 고유한 브라우저 쿠키다
구글 애널리틱스가 사용자를 인지하는 기준은 고유한 브라우저 쿠키 정보입니다. 브라우저 쿠키를 삭제하면 특정 사용자의 방문 이력도 삭제됩니다. 만약 특정 사용자가 쿠키가 삭제된 상태에서 동일 브라우저로 재방문을 할 경우 GA는 해당 사용자를 신규 방문자로 인지합니다.  
쿠키는 기기의 브라우저 단위로 발행됩니다. 동일 기기라도 인터넷 익스플로러와 크롬 브라우저의 쿠키 정보는 다릅니다. 하지만 크롬 시크릿 브라우저로 접속하는 방문자의 경우, 해당 세션 이후에는 쿠키 정보가 자동으로 삭제됩니다.  
일반적으로 방문자 중 30%는 한 달에 한 번은 쿠키를 삭제한다는 통계가 있습니다. GA쿠키는 방문자가 삭제하지 않는 이상 2년 동안 지속됩니다.  
만약 브라우저와 관계없이 로그인 사용자를 분석하고 싶다면 GA의 user-ID 기능을 활용. 사용자가 로그인시 고유한 Key값을 받아서 교차 기기 행동 데이터를 GA에서 확인할 수 있습니다.  
GA는 순쿠키 횟수를 카운팅하므로 동일한 사람이 7번 방문해도 1로 카운팅 됩니다. 그래서 다른 분석 도구와 사용자 지표를 비교했을 때, GA가 낮게 나옵니다.

### 세션 지표, 제대로 알고 쓰자
세션은 웹 또는 앱 접속 후 처음으로 히트가 발생하는 순간 카운팅됩니다. 히트라 함은 이벤트 또는 페이지뷰가 될 수 있습니다. 세션은 최초 카운팅부터 30분간 아무런 액션이 없을 경우 종료됩니다.  
일반적으로 서비스 트래픽상 세션이 사용자보다 높고 페이지별 트래픽상 사용자가 세션보다 높다.  왜냐하면 세션은 랜딩 페이지에서만 카운팅되기 때문에, 만약 A에 랜딩하고 B로 이동 후 종료하면 A 페이지에서만 세션이 카운팅 됩니다.  
추가로 세션은 클릭 수가 아닙니다. 클릭 수는 페이지뷰 또는 이벤트 지표에 가깝습니다. 특정 페이지의 종료율은 세션 종료 전 마지막으로 조회된 페이지라고 이해해야 합니다.  
* GA 세션 종료 케이스
1. 30분 동안 아무런 액션이 없을 때
2. 자정 밤 12시가 지날 때
3. 다른 캠페인으로 웹사이트에 랜딩 될 때
<br>
<br>

### 세션 시간이 0으로 나오는 이유
GA는 마지막 페이지에 머문 시간을 무시합니다. 그래서 대부분 세션 시간이 0으로 확인된 경우는 세션이 1일 때가 많습니다. 마지막 페이지에서 아무리 많은 시간을 머물러도 GA는 마지막 히트 전 시간만 합하여 세션 시간으로 계산합니다.  
GA는 단일 페이지에서 머문 시간을 측정하지 않습니다. 그래서 특정 페이지에서 25분을 머무르다가 바로 이탈해도 세션 시간은 0으로 측정됩니다.  
만약 마지막 페이지 머문 시간을 세션 시간에 포함하려면 GTM을 통해, 일정 시간 체류시 이벤트를 강제로 발생시키는 방법이 있습니다. 그러면 이전 대비 전체 이탈률이 다소 하락합니다.  
<br>
<br>

### 이탈률을 낮춰야 전환율이 개선됩니다
GA에서 말하는 이탈률의 정확한 의미는 '단일 히트 세션'입니다. 세션(방문)에서 히트가 1회만 발생했다는 의미입니다 - 다른 페이지를 탐색하지 않고 이탈한 경우. 웹사이트 평균 이탈률은 보통 보수적으로 잡아도 40%입니다.  
하지만 프로모션 페이지 내 CTA버튼 클릭, 외부 도메인 연결, 콘텐츠 5분간 읽고 웹사이트 종료 등의 경우도 GA는 이탈한 사용자라고 간주합니다. 왜냐하면 GA는 클릭이나 상세 활동을 판단하지 못하므로, GTM으로 각 조건을 만족시킬 때 GA로 이벤트 전송하여 이탈로 집계되지 않도록 설정이 필요합니다.  
<br>
<br>

### 전환율은 랜딩 페이지에 달려 있다
세션의 시작 단계이므로 랜딩 페이지 이탈을 막아야 전환을 높일 수 있습니다. 특정 랜딩 페이지 이탈률이 높다면 해당 페이지 광고 소재를 개선해야 합니다. 유입은 누구나 할 수 있지만 이탈을 최소화해야 합니다.  
서비스의 이탈률은 전체 세션을 기반으로 계산됩니다. 이는 각 랜딩 페이지의 이탈률이 전체 이탈률에 영향을 준다는 의미입니다.  
일반적으로 배너가 키워드 광고보다 이탈률이 높아 광고 단가도 키워드보다 낮습니다. 배너 광고만 보고 바로 구매하는 경우는 흔치 않기 때문에, 배너 광고의 성과는 브랜드 키워드 쿼리가 얼마나 증가했는지로 판단해야 합니다.

### 데이터는 쪼개야 인사이트가 나온다
'모든 고객을 대상으로 물건을 팔아서는 안 된다. 아무에게도 팔지 않겠다는 얘기다. BY 피터 드러커'
세그먼트를 만들 때는 너무 소수를 대표하는 세그먼트를 만들면 안 됩니다. 임의로 '일반/관심/충성/열성'으로 나눠, 낮은 등급 고객을 상위 등급으로 올리기 위해 문제가 무엇인지 파악하는 것만 알면 분석의 절반은 끝난 것이나 다름없습니다.  
쇼핑몰에서 가장 중요한 고객은 바로 충성 고객입니다. 이들의 평균 구매액을 늘리는 것이 마케팅 활동의 핵심이 되어야 합니다. 세그먼트를 이용한 리마케팅을 진행하면 이들과 유사 행동을 보이는 고객을 구글 애드워즈 광고에 모수로 활용하면 당연히 일반 잠재고객 대비 광고에 대한 반응이 높게 나올 것입니다. 예를 들어, 프로모션 페이지 방문자 중 스크롤 50%와 80% 내린 고객에게 다른 배너 광고를 보여주는 광고를 할 수 있습니다.

### 캠페인 링크 생성시 주의점
캠페인이란 특정 목적을 가지고 운영되는 마케팅 활동으로, GA에서 수집하려면 링크에 별도 UTM 변수를 붙여줘야 합니다.  
링크 뒤에 붙이는 필수 캠페인 변수는 3가지로, 이 중 하나라도 없으면 캠페인 보고서에 데이터가 수집되지 않습니다. 변수 네이밍 정답은 없으나 효율적인 운영을 위해 네이밍에 대한 룰을 정하는 것이 필요합니다.  
> * (필수) utm_source(소스명) = 출처(Where)
> * (필수) utm_medium(매체명) = 매체 및 광고 형태(How)
> * (필수) utm_campaign(이름) = 목적(What)
> + (추가) utm_term(키워드명) = 유입된 키워드
> + (추가) utm_content(광고 콘텐츠) = 콘텐츠 컨셉
<br>

### 캠페인 유지 기간이 중요한 이유
직접 방문 이전에 캠페인을 통한 방문 이력이 있으면 GA는 Direct 방문이 아닌 캠페인 방문으로 성과를 먼저 부여합니다. 캠페인 방문 뒤 3개월 후 직접 유입시 Direct가 아니라 캠페인 성과로 부여되는 프로세스입니다. 판매 주기가 아무리 길어도 연관성이 떨어지기 때문에 3개월을 넘기면 안됩니다.  
GA 캠페인 유지 기간은 기본적으로 6개월로, 속성 설정의 세션 메뉴에서 최소 1분에서 24개월까지로 조정할 수 있습니다. 참고로 네이버 광고 성과 유지 기간은 7~20일이므로, 네이버 의존도가 높을 경우 유지 기간을 1개월로 수정하면 네이버 광고 시스템 성과와 GA간 매체 차이를 줄일 수 있습니다.

### 맞춤형 채널 그룹을 설계하는 방법
'보기 설정 > 채널 설정 > 채널 그룹 메뉴'에서 특정 소스/매체를 채널로 분류 가능합니다. 다만, 채널 그룹이 상단에 위치할수록 트래픽 분류에 우선순위를 가지게 됩니다. 채널은 기본 채널과 신규 채널 그룹이 있는데, 기본 채널 그룹은 생성 시점부터 데이터가 분류되고 신규 채널 그룹은 생성 이전 데이터도 소급 적용되는 장점이 있습니다. 또한 기본 채널 그룹은 데이터 샘플링이 걸리지 않지만 신규 채널 그룹은 샘플링에 영향을 받습니다.  
cf. 채널별 데이터 분류 기준 (https://support.google.com/analytics/answer/3297892?hl=ko)


### 구글 오가닉 검색어 확인하는 방법
구글에서 검색 광고가 아닌 자연 검색으로 유입되는 경우 키워드 값이 (not provided)로 표현됩니다. 이를 확인하기 위해서는 구글 서치 콘솔을 등록해야 합니다. 서치 콘솔 데이터 연동시 '획득>서치 콘솔'메뉴에서 데이터를 확인할 수 있습니다. 구글 서치 콘솔에 등록하면 '총 클릭수/총 노출수/평균 CTR/평균 게재순위'를 확인할 수 있지만, 각 키워드별 전환율 및 목표 달성 여부까지는 확인되지 않습니다.
브랜드 인지도가 클수록 브랜드 키워드의 양이 많아집니다. 만약 구글 검색 결과에 노출이 많이되고 게재 순위가 낮은 경우라면 해당 키워드는 CPC 단가를 높여서 클릭 유도하는 전략이 필요합니다.
cf. 국내에서는 SEO(검색 엔진 최적화)의 중요성을 인지하지 못하는 경우가 많은데, SEO 기본만 지켜도 광고비를 획기적으로 줄일 수 있습니다. 그 중 하나가 서치 콘솔 등록입니다.



### 필터 설계에 관한 다양한 팁
필터는 정확한 데이터를 수집하기 위해 적용합니다. 참고로 필터는 적용 이후 누적 데이터에만 영향을 받을 뿐, 이전 데이터에는 적용이 되지 않습니다. 또한 필터가 다수일 경우 필터가 생성되는 시간 순서대로 적용이 됩니다. 필터는 '보기 설정' 메뉴에서 조작 가능합니다.
* 필터가 적용되는 경우
- 업무 유관자(내부 직원 및 대행사) 및 봇(Bot) 트래픽 제외
- 특정 도메인 혹은 기기별 트래픽 보기를 만들 때
- 수집되는 데이터 패턴을 조작할 때



### 웹사이트 내부 검색어 확인 방법
고객의 관심사를 즉각적으로 확인할 수 있는 0순위 데이터는 내부 검색어입니다. 계절 및 이벤트의 영향을 많이 받는데, 내부 검색어 트렌드 추이와 검색엔진에서 유관 키워드 검색 추이를 살피며 고객 니즈에 빠르게 대응해야 합니다.
GA에서 키워드와 검색어는 다른 개념입니다. 키워드는 웹사이트 유입시 검색한 단어이고, 검색어는 웹사이트 내 검색창에서 검색한 단어를 의미합니다. 보통 링크 주소에 SearchQuery 항목으로 보여지는데, 검색해도 변수가 확인되지 않는 경우는 개발자가 웹사이트 개발시 검색어 변수를 숨김 처리했기 때문입니다.(Post 방식) 이 경우에는 GTM으로 가상 페이지뷰를 만들어, 해당 페이지에서 검색어 변수 입력시 검색어 보고서에서 해당 데이터를 보여주도록 설정해야합니다.
검색 기능 경험자 세그먼트로 비교해보면 일반적으로 내부 검색 기능을 이용한 사용자는 그렇지 않은 사용자 대비 평균 전환율과 구매액이 높습니다. 그러므로 '추천 검색어'와 '연관 검색어'로 방문 고객들이 검색을 하도록 유도하는 전략이 필요합니다.


### 기획에서부터 분석을 고려해야 한다
데이터 분석을 고려한 홈페이지 구축시 결과적으로 많은 이득이 있습니다.
1. 페이지 제목과 경로(url)는 분석의 중요한 재료다
장바구니 페이지 제목 '장바구니', 경로 URL에 'cart'라는 단어를 붙이는 경우가 많은데, 이는 검색 순위가 상위에 위치하도록 도움을 줍니다. 이를 '검색엔진 최적화(SEO)'라고 합니다.
페이지를 생성할 때 계층 구조 설계만 잘해도 광고에 사용되는 비용을 절감할 수 있습니다. 다시 말해, 제목과 경로만으로도 방문자가 보고 있는 페이지의 컨셉과 목적을 알 수 있어야 합니다. 
2. 배너 이미지 태그에 Alt(대체 텍스트) 속성 값을 붙여주자
랜더링 실패시 이미지 대신 문자열이 나오도록 설정된 것이므로, 웹 접근성 준수를 위해서도 반드시 필요한 것입니다. 또한 분석적으로 접근하자면 alt 값은 방문자가 이미지 클릭시 어떤 그림을 클릭했는지 데이터를 수집할 수 있게 합니다.
3. 동일한 서비스 확장이라면 서브 도메인을 생성하자
